Execute HP-OO v10 flows from vRealize Orchestrator

23 February 2018
VMware / vRealize Orchestrator / REST / HP-OO / HP

Have you used the HTTP REST plugin in vRealize Orchestrator (vRO) before? It’s a fantastic plugin that really pushes the envelope on what you can integrate with vRO. Here’s a quick rundown of how you configure your REST operations:

  1. Configure a REST endpoint.
  2. Define a REST operation against the specified endpoint.
  3. Create a workflow from the new REST operation to use in your workflows.

These steps are easy to do, but following this method only support a handful of authentication methods and you can’t define additional HTTP headers in your REST operations when you create them. Coincidentally, HP-OO requires the custom header “X-CSRF-TOKEN” to be sent when you make a POST against the HP-OO server.

What I’ll cover in this post:

  1. Creating a HP-OO REST endpoint in vRO
  2. Defining a REST operation in vRO to perform a GET against HP-OO and receive a X-CSRF-TOKEN.
  3. Defining a second REST operation in vRO to perform a POST against HP-OO to execute a workflow.
  4. Generating the workflows from the REST operations in vRO to get us some editable workflows of the defined REST operations.
  5. Modifying the generated workflows to pass the required token between them.
  6. Modifying the generated workflow to add the X-CSRF-TOKEN to the header of the POST method.

I’ll assume that you’ve already got a HP-OO instance running with the privileges and configuration changes made to allow REST calls.

Creating a HP-OO REST endpoint in vRO

In the vRO client, switch to the Design context and navigate to the Library->HTTP-REST folder.

//prepare request
//Do not edit 
var inParamtersValues = [];
var request = restOperation.createRequest(inParamtersValues, null);
//set the request content type
request.contentType = "";
System.log("Request: " + request);

//Customize the request here

contentLength = response.contentLength;
headers = response.getAllHeaders();
